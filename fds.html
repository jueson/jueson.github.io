友情链接强制换行解决方案

展示如何通过CSS修改使列表项强制换行显示，同时提供原始布局与修改后的布局对比

原始布局

  • GitHub
  • Twitter
  • LinkedIn
  • YouTube
  • Instagram
  • Facebook

原始布局使用 display: inline-block，所有链接在同一行显示，超出容器宽度时会换行但布局不整齐。

解决方案效果

  • GitHub
  • Twitter
  • LinkedIn
  • YouTube
  • Instagram
  • Facebook
  • Pinterest
  • TikTok
  • Discord
  • Reddit

通过设置 display: block，每个链接独占一行，整齐排列并带有悬停效果。

实现说明

要使友情链接强制换行显示，只需修改CSS中的 display 属性：

/* 原始样式 - 内联显示 */
.list-inline li {
  display: inline-block;
}

/* 修改后样式 - 块级显示（强制换行） */
.solution-list li {
  display: block; /* 关键修改 */
  margin-bottom: 12px; /* 添加间距 */
}

在Jekyll模板中，只需确保列表项具有 display: block 样式即可：

<ul class="solution-list">
  {% for friend in site.friends %}
    <li><a href="{{friend.href}}">{{friend.title}}</a></li>
  {% endfor %}
</ul>

/* 对应的CSS */
.solution-list li {
  display: block;
  margin-bottom: 12px;
}

这种方法保持了模板的简洁性，同时通过CSS实现了强制换行的效果，并添加了美观的样式增强用户体验。
